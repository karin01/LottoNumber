# 로또 확률 기반 번호 생성기 — 개발 기록 (옵시디언)

## 목적

단순 랜덤이 아니라 **1회차부터의 로또 당첨 기록**을 반영해 **확률을 계산**하고, 그 확률에 따라 번호를 생성하는 생성기.

## 구조 요약

| 파일 | 역할 |
|------|------|
| `lotto_data.py` | 동행복권 API로 회차별 당첨 번호 수집, `lotto_history.json` 캐시. API 실패 시 로컬 캐시만 사용 |
| `lotto_probability.py` | 번호별 출현 횟수·비율(확률) 계산. 전체 회차 / 최근 N회 가중 선택 가능 |
| `lotto_generator.py` | 확률 분포로 6개 번호 비복원 추출 (가중 랜덤) |
| `main.py` | CLI: `fetch` / `stats` / `generate` |

## 확률 계산 방식

1. **전체 회차**: 1회~마지막 회차까지 본당첨 6개 번호만 집계 → 출현 횟수 → 합 1로 정규화해 확률로 사용.
2. **최근 N회 가중**: 최근 N회만 사용하고, 최근일수록 가중치를 높게 부여한 점수로 동일하게 정규화.

생성 시에는 이 확률을 가중치로 해서 `random.choices`로 6개 뽑고, 중복 제거 후 6개가 되도록 보정한 뒤 오름차순 정렬.

## 데이터 출처

- **동행복권 비공식 API**: `https://www.dhlottery.co.kr/common.do?method=getLottoNumber&drwNo={회차}`
- 응답 필드: `drwtNo1`~`drwtNo6`, `bnusNo`, `drwNo`, `drwNoDate`, `returnValue` 등.
- API 접속이 막히는 환경에서는 기 저장된 `lotto_history.json` 또는 `lotto_history_sample.json`(1~5회차 샘플) 사용.

## 사용 예시

```bash
# 당첨 기록 수집 (최초 1회 권장)
python main.py fetch

# 확률 통계 (상위 15개 번호)
python main.py stats
python main.py stats --recent 50

# 확률 기반 번호 생성
python main.py generate --count 5
python main.py generate --count 10 --recent 100 --seed 42
```

## 주의사항

- 과거 출현 빈도는 **미래 당첨 확률을 보장하지 않음**. 참고·재미용.
- 동행복권 API는 비공식이며 요청 제한·변경 가능성 있음.
