# 작업 기록: 주간 당첨 요약이 최신 회차로 안 갱신되던 문제

**날짜:** 2026-04-13

## 현상

- 토요일 추첨(예: 4월 18일) 이후에도 화면에 **이전 주 회차**(예: 1219회, 4월 11일)만 표시됨.
- 데이터 출처가 `뉴스 RSS(추정)`일 때 특히 그런 경향.

## 원인 (WHY)

1. **동행복권 JSON API**가 호스팅(Render 등)에서 막히면, 주간 요약은 **로컬 `lotto_history.json`의 가장 큰 회차**만 사용했음 → 파일이 1219까지면 항상 1219.
2. 뉴스 RSS 조회 시 `draw_no=None`이면 **역시 파일 최대 회차**로만 검색해서, 1220회 기사를 찾지 않음.
3. `_guess_current_draw_no()`가 **서버 로컬 날짜(UTC)**만 쓰면 한국 토요일/일요일 경계에서 주차가 어긋날 수 있어, **KST 기준 날짜**로 통일함.

## 수정 요약

- `lotto_data.get_latest_draw_no()`: 동행복권 **메인 HTML**의 최신 회차를 우선 사용(가능할 때).
- API 실패 후 표시 회차: `공식 메인 최신`이 있으면 그 회차, 없으면 `max(로컬 최대, KST 추정 회차)`.
- `_fetch_weekly_summary_from_news`: 위에서 정한 회차로 RSS 검색 + `parse_round_from_news_text`로 **제목 속 당첨번호** 보강.
- `_fetch_lotto_raw`: `REQUEST_HEADERS`로 API 차단 완화 시도.
- 메인 파싱도 실패·번호 확보 불가 시에만, 예전처럼 **파일 최신 회차**로 폴백(빈 응답 방지).

## 추가 (2026-04-13 이후)

- `get_latest_draw_no()`(동행 메인 HTML)이 **여전히 1219**만 돌려주는 환경이 있어, `target_no`를 **공식값만** 쓰면 1219에 고정됨.
- 해결: `_expected_latest_announced_draw_no_kst()`로 **KST·토요일 21:15 기준 ‘발표된 마지막 회차’**를 구해 `target_no = max(로컬, 공식, 달력)`으로 병합.

## lt645 공식 추첨결과 API (추가)

- `https://www.dhlottery.co.kr/lt645/result` 페이지는 브라우저용이며, 실제 데이터는 **`selectPstLt645Info.do?srchLtEpsd=회차`** JSON으로 내려옴.
- `common.do?method=getLottoNumber`이 IP에서 막혀도 위 API는 동작하는 경우가 있어, `lotto_data._fetch_one_draw` → `fetch_lotto_draw_raw`에서 **폴백**으로 연동함.

## 크롤링·재시도 보강

- lt645: **재시도**(기본 3회, `REQUEST_DELAY_SEC` 간격), **`srchDir=latest`**로 최신 한 건과 요청 회차 일치 시 사용, **`/lt645/result` HTML 선요청** 후 API 재시도(브라우저 순서 유사).
- `get_latest_draw_no`: 메인 HTML 대신 **`srchDir=latest` JSON의 `ltEpsd`** 우선(지연·캐시로 메인이 1219에 머물 때 대비).
- `getLottoNumber`: 2회 재시도, lt645 전체 실패 후 페이지 워밍 뒤 **한 번 더** getLottoNumber 시도.

## 관련 코드

- `lotto_data.py`: `_fetch_one_draw_lt645`, `fetch_lotto_draw_raw`
- `server.py`: `_get_weekly_summary`, `_fetch_weekly_summary_from_news`, `_guess_current_draw_no`, `_expected_latest_announced_draw_no_kst`, `_fetch_lotto_raw`

## 보조 표: «목록에 없습니다» 오해 (2026-04-13)

- **현상:** 상단에서 예) 1015회 당첨번호는 보이는데, 아래 표에는 «선택한 1015회는 현재 목록에 없습니다»가 뜸.
- **원인:** `/api/draws?limit=200`은 **최신 200회**만 내려줌. 1220 기준으로 1015회는 그 범위 밖. 반면 `/api/draw_by_no`는 **전체 `lotto_history` 캐시**에서 찾음.
- **1차 수정:** `applyDrawNoView`에서 조회 성공 시 해당 회차를 `__CURRENT_DRAWS__`에 **병합**해 표가 주변 회차와 함께 보이게 함. 범위 밖일 때 안내 문구를 위 원인에 맞게 수정하고, 무의미한 `refreshDrawsTable()` 재호출 제거.
- **2차 보정(같은 날):** 1회·5회 같은 오래된 회차 조회 시 하단 표가 `1000, 5, 1`처럼 섞여 보이는 부작용이 확인됨.  
  해결: `__RECENT_DRAWS_BASE__`(서버에서 받은 최신 200회 원본)를 별도로 유지하고, 조회 회차가 최근 범위 밖이면 하단 표에는 병합하지 않고 **최근 목록 그대로 유지**하도록 변경.

## 선택 회차 기준 반복 번호 분석 (2026-04-23)

- **요청:** 선택 회차 기준으로 과거 N개 회차(기본 5개)를 묶어, 반복되는 번호가 있으면 번호별 출현 횟수를 보여달라는 요구.
- **수정(UI):** `drawNoInput` 옆에 `drawRepeatWindowInput`(2~30, 기본 5) 추가. 단건 조회 성공 시 `drawRepeatResult` 영역에 분석 결과 표시.
- **수정(API):** `/api/draw_window_summary?start=회차&count=개수` 추가.
  - `start`부터 과거 방향으로 최대 `count`개 회차를 수집
  - 본번호 6개 기준 번호 출현 횟수 집계
  - `2회 이상` 나온 번호를 `repeated_numbers`로 반환
- **표시 방식:** 반복 번호가 없으면 “동일 번호 반복 없음”을 표시하고, 있으면 `번호 + 출현 횟수`를 배지 형태로 출력.

## 반복 분석 기준 개선: 지정 회차 vs 이전 N회 (2026-04-23)

- **요청:** 1220회 분석 시 `1220`과 `1219~1215`를 비교하는 방식으로 바꿔 달라는 요구.
- **기존 문제:** 초기 구현은 `1220~1216` 전체 구간에서 “서로 중복”을 계산해, 사용자가 기대한 “기준 회차(1220) 번호가 이전 회차에서 얼마나 반복됐는지”와 의미가 달랐음.
- **개선 내용(API):** `/api/draw_window_summary`를 기준 회차(`base_draw`)와 이전 비교 회차(`compare_draws`)로 분리.
  - `start` 회차는 비교의 기준으로만 사용
  - 비교 대상은 `start-1`부터 과거 `count`개
  - 결과는 기준 회차 번호 중 이전 구간에서 다시 나온 번호만 `matched_numbers`로 반환
- **개선 내용(UI):** 문구를 “이전 비교 구간”으로 명확히 변경하고, “제1220회 번호를 이전 5개 회차와 비교” 형태로 안내하도록 수정.

## 데이터 헬스 자동 경고 토스트 (2026-04-23)

- **요청:** 헬스 카드 기준으로 자동 경고 토스트를 띄워 달라는 요청.
- **조건:** 아래 중 하나라도 만족하면 경고 토스트 표시.
  - `latestLag >= 3` (최신 회차 지연 3회차 이상)
  - `missingInRecent30 >= 2` (최근 30회 결측 2개 이상)
- **보정:** 같은 조건에서 새로고침할 때 경고가 매번 반복되지 않도록 `lastHealthWarningKey`로 중복 노출 방지.

## 경고 토스트 클릭 시 헬스 카드 이동 (2026-04-23)

- **요청:** 경고 토스트를 누르면 헬스 카드로 바로 이동하게 해 달라는 요청.
- **수정:** `showToast(message, type, targetId)` 형태로 확장하고, 토스트 클릭 시 `targetId`가 있으면 해당 카드로 `smooth scroll`.
- **적용:** 데이터 헬스 경고 토스트는 `cardDataHealth`를 타겟으로 지정.

## 경고 토스트 버튼형 강조 (2026-04-23)

- **요청:** 헬스 경고 토스트가 클릭 대상임을 더 명확히 보여달라는 요청.
- **수정:** `targetId`가 있는 토스트에 `상세 보기` 뱃지(`toast-action`)를 표시.
- **효과:** 일반 알림과 구분되며, 사용자가 “이 토스트는 눌러서 이동 가능”함을 즉시 인지.
