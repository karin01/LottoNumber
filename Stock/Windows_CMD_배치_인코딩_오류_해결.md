# Windows CMD 배치 실행 시 `'??'`, `nobreak` 등 알 수 없는 명령 오류

## 원인 (WHY)

- 프로젝트의 `.bat` 파일이 **UTF-8(무 BOM)** 으로 저장되어 있었음.
- 기본 코드 페이지가 **CP949** 인 `cmd.exe`는 파일 시작 부분과 한글·특수문자를 **한 바이트씩** 잘못 끊어 읽을 수 있음.
- 그 결과 `timeout /t 3 /nobreak` 같은 한 줄이 깨져 **`nobreak`만 별도 명령**처럼 실행되는 등, 화면에 이상한 문자 조각이 명령으로 인식됨.

## 조치 (2026-04-14)

- `Stock` 폴더의 모든 `*.bat`을 **UTF-8 BOM** + **CRLF** 로 다시 저장함.
- `cmd`가 파일을 UTF-8로 인식해 한 줄이 통째로 해석되도록 함.

## 대안

- 배치 대신 **PowerShell**: `run_stock.ps1` 은 의도적으로 ASCII-only 로 두어 인코딩 문제를 피함.
- 메모장에서 수동 저장 시: **다른 이름으로 저장 → 인코딩: UTF-8(서명 있음)** 선택.

## 리스크

- 구형 Windows/특수 환경에서는 여전히 `chcp 65001` 과 조합 시 이슈가 있을 수 있음. 그때는 배치 내 한글을 줄이거나 PowerShell 실행을 우선 검토.
