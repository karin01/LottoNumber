# 라틴댄스 파티 웹앱 호스팅 가이드

## 옵션 1: Netlify (무료, 추천)

### 1단계: GitHub에 코드 업로드
1. GitHub 계정 생성 (없다면)
2. 새 저장소 생성: `latin-dance-party`
3. 로컬 코드를 GitHub에 업로드:
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/사용자명/latin-dance-party.git
git push -u origin main
```

### 2단계: Netlify 배포
1. https://netlify.com 접속
2. "Sign up" → GitHub 계정으로 로그인
3. "New site from Git" 클릭
4. GitHub 선택 → `latin-dance-party` 저장소 선택
5. 배포 설정:
   - Build command: 비워두기 (정적 사이트)
   - Publish directory: `static`
6. "Deploy site" 클릭

### 3단계: 도메인 설정
1. Netlify 대시보드에서 "Domain settings" 클릭
2. "Add custom domain" 클릭
3. `latinmat.co.kr` 입력
4. DNS 설정 안내에 따라 도메인 제공업체에서 CNAME 설정

## 옵션 2: Vercel (무료)

### 1단계: Vercel 배포
1. https://vercel.com 접속
2. GitHub 계정으로 로그인
3. "New Project" 클릭
4. `latin-dance-party` 저장소 선택
5. Framework Preset: "Other" 선택
6. "Deploy" 클릭

### 2단계: 도메인 연결
1. Vercel 대시보드에서 "Settings" → "Domains"
2. "Add Domain" 클릭
3. `latinmat.co.kr` 입력
4. DNS 설정 안내에 따라 설정

## 옵션 3: GitHub Pages (무료)

### 1단계: GitHub Pages 활성화
1. GitHub 저장소에서 "Settings" 탭 클릭
2. 왼쪽 메뉴에서 "Pages" 클릭
3. Source: "Deploy from a branch" 선택
4. Branch: "main" 선택
5. Folder: "/docs" 또는 "/static" 선택
6. "Save" 클릭

### 2단계: 도메인 연결
1. "Custom domain" 섹션에서 `latinmat.co.kr` 입력
2. "Save" 클릭
3. DNS 설정 안내에 따라 설정

## Firebase 설정 (모든 옵션 공통)

### 1단계: Firebase 콘솔 설정
1. https://console.firebase.google.com 접속
2. 프로젝트 선택 또는 새 프로젝트 생성
3. "Authentication" → "Settings" → "Authorized domains"
4. 다음 도메인들 추가:
   - `latinmat.co.kr`
   - `your-app.netlify.app` (Netlify 사용 시)
   - `your-app.vercel.app` (Vercel 사용 시)

### 2단계: Firebase 설정 파일 업데이트
1. `static/js/firebase-config.js` 파일에서 도메인 확인
2. 필요시 Firebase 프로젝트 설정 업데이트

## DNS 설정 (도메인 제공업체에서)

### CNAME 레코드 추가:
- Name: `@` 또는 `www`
- Value: 
  - Netlify: `your-app.netlify.app`
  - Vercel: `your-app.vercel.app`
  - GitHub Pages: `사용자명.github.io`

## 배포 후 확인사항

1. **Firebase 인증 확인**: 로그인 기능이 정상 작동하는지 확인
2. **이미지 업로드 확인**: 갤러리 기능이 정상 작동하는지 확인
3. **반응형 디자인 확인**: 모바일에서도 정상 표시되는지 확인
4. **성능 확인**: 페이지 로딩 속도 확인

## 문제 해결

### Firebase 오류 해결:
- 도메인이 Firebase Authorized domains에 등록되었는지 확인
- Firebase 설정 파일의 프로젝트 ID가 올바른지 확인

### 이미지 404 오류 해결:
- 유튜브 썸네일 URL이 올바른지 확인
- 로컬 이미지 파일 경로가 올바른지 확인

### SSL 인증서 문제:
- 대부분의 호스팅 서비스에서 자동으로 SSL 제공
- 필요시 수동으로 SSL 인증서 설정
