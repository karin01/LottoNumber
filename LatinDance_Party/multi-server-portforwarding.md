# 🌐 멀티 서버 포트포워딩 설정 가이드

## 📋 여러 서버 동시 운영 방법

### 1. 포트 번호 분배

각 서버마다 다른 포트 번호를 사용합니다:

```
라틴댄스 파티 서버: 포트 8000
다른 앱 서버: 포트 8001
세 번째 앱 서버: 포트 8002
...
```

### 2. 포트포워딩 설정

#### 공유기 관리자 페이지에서 포트포워딩 규칙 추가:

```
규칙 1: 외부 포트 8000 → 내부 IP [PC IP] → 내부 포트 8000 (라틴댄스 파티)
규칙 2: 외부 포트 8001 → 내부 IP [PC IP] → 내부 포트 8001 (다른 앱)
규칙 3: 외부 포트 8002 → 내부 IP [PC IP] → 내부 포트 8002 (세 번째 앱)
```

### 3. 방화벽 설정

#### Windows 방화벽에서 포트 허용:

```
포트 8000: 라틴댄스 파티 서버
포트 8001: 다른 앱 서버
포트 8002: 세 번째 앱 서버
```

### 4. 서버 시작 방법

#### 방법 1: 개별 배치 파일 사용
```bash
# 라틴댄스 파티 서버
server-start-latin-dance.bat

# 다른 앱 서버 (새 터미널에서)
server-start-other-app.bat
```

#### 방법 2: 멀티 서버 관리자 사용
```bash
python multi-server-manager.py
```

### 5. 접속 주소

#### 로컬 접속:
```
라틴댄스 파티: http://localhost:8000
다른 앱: http://localhost:8001
세 번째 앱: http://localhost:8002
```

#### 외부 접속:
```
라틴댄스 파티: http://[외부IP]:8000
다른 앱: http://[외부IP]:8001
세 번째 앱: http://[외부IP]:8002
```

### 6. 서버 관리 도구

#### 멀티 서버 관리자 기능:
- 모든 서버 한 번에 시작/중지
- 개별 서버 시작/중지
- 서버 상태 실시간 확인
- 접속 주소 자동 표시

#### 사용법:
```bash
python multi-server-manager.py
```

메뉴에서 원하는 작업 선택:
1. 모든 서버 시작
2. 모든 서버 중지
3. 서버 상태 확인
4. 특정 서버 시작
5. 특정 서버 중지

### 7. 성능 고려사항

#### PC 사양에 따른 권장 서버 수:
- **저사양 PC**: 2-3개 서버
- **중사양 PC**: 3-5개 서버
- **고사양 PC**: 5개 이상 서버

#### 메모리 사용량 (대략적):
- Python HTTP 서버: 50-100MB/서버
- Node.js 서버: 100-200MB/서버
- PHP 서버: 30-80MB/서버

### 8. 문제 해결

#### 포트 충돌 문제:
```bash
# 포트 사용 중인 프로세스 확인
netstat -ano | findstr :8000
netstat -ano | findstr :8001

# 프로세스 종료
taskkill /PID [프로세스ID] /F
```

#### 서버 시작 실패:
1. 포트 번호 변경
2. 방화벽 설정 확인
3. 다른 프로그램과의 충돌 확인

### 9. 보안 설정

#### 각 서버별 보안:
- 필요한 포트만 열기
- 정기적인 보안 업데이트
- 로그 모니터링
- 백업 시스템 구축

### 10. 모니터링

#### 서버 상태 모니터링:
```python
# 각 서버별 상태 확인
import requests

servers = {
    "라틴댄스 파티": "http://localhost:8000",
    "다른 앱": "http://localhost:8001"
}

for name, url in servers.items():
    try:
        response = requests.get(url, timeout=5)
        print(f"{name}: {'정상' if response.status_code == 200 else '오류'}")
    except:
        print(f"{name}: 연결 실패")
```
