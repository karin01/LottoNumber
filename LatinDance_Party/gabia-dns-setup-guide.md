# 가비아 DNS 설정 가이드 (Firebase Hosting)

## 1. 가비아 관리자 페이지 접속

### 로그인
1. https://www.gabia.com 접속
2. 우측 상단 "로그인" 클릭
3. 가비아 계정으로 로그인

### 도메인 관리
1. "도메인 관리" 메뉴 클릭
2. "latinmat.co.kr" 도메인 찾기
3. "관리" 버튼 클릭

## 2. DNS 관리 페이지 접속

### DNS 설정
1. "DNS 관리" 또는 "네임서버 관리" 클릭
2. "DNS 레코드 관리" 선택
3. "레코드 추가" 또는 "새 레코드" 버튼 클릭

## 3. A 레코드 추가

### 첫 번째 레코드
```
레코드 유형: A
호스트명: @ (또는 latinmat.co.kr)
값: 199.36.158.100
TTL: 3600
우선순위: (비워두기)
```

### 저장 후 두 번째 A 레코드 추가
```
레코드 유형: A
호스트명: @ (또는 latinmat.co.kr)
값: 151.101.1.195
TTL: 3600
우선순위: (비워두기)
```

### 세 번째 A 레코드 추가
```
레코드 유형: A
호스트명: @ (또는 latinmat.co.kr)
값: 151.101.65.195
TTL: 3600
우선순위: (비워두기)
```

## 4. TXT 레코드 추가

### 소유권 확인용 TXT 레코드
```
레코드 유형: TXT
호스트명: @ (또는 latinmat.co.kr)
값: hosting-site-share-note-ef791
TTL: 3600
```

## 5. CNAME 레코드 추가 (선택사항)

### www 서브도메인
```
레코드 유형: CNAME
호스트명: www
값: latinmat.co.kr
TTL: 3600
```

## 6. 설정 확인

### 레코드 목록 확인
추가된 레코드들이 다음과 같이 표시되어야 합니다:

```
A    @    199.36.158.100
A    @    151.101.1.195
A    @    151.101.65.195
TXT  @    hosting-site-share-note-ef791
CNAME www latinmat.co.kr
```

## 7. DNS 전파 대기

### 전파 시간
- DNS 변경사항이 전파되는데 **최대 24-48시간** 소요
- 보통 **1-2시간 내**에 적용됨
- 글로벌 DNS 캐시로 인해 지역별로 다를 수 있음

### 확인 방법
1. https://dnschecker.org 에서 확인
2. `latinmat.co.kr` 검색
3. A 레코드가 올바른 IP로 설정되었는지 확인

## 8. Firebase에서 확인

### 도메인 소유권 확인
1. Firebase Console → Hosting → Custom domains
2. "latinmat.co.kr" 도메인 상태 확인
3. "소유권 확인됨" 또는 "DNS 확인됨" 표시 확인

### SSL 인증서 발급
- 소유권 확인 후 자동으로 SSL 인증서 발급
- 발급까지 **최대 24시간** 소요

## 9. 문제 해결

### DNS 전파가 안 될 때
1. 브라우저 캐시 삭제
2. 다른 네트워크에서 테스트
3. DNS 캐시 플러시 (Windows: `ipconfig /flushdns`)

### 레코드 추가가 안 될 때
1. 기존 A 레코드가 있다면 삭제 후 추가
2. TTL 값을 300으로 설정
3. 가비아 고객센터 문의

### Firebase에서 인식 안 될 때
1. DNS 전파 완료 대기 (최대 48시간)
2. TXT 레코드 정확성 확인
3. Firebase Console에서 "재확인" 버튼 클릭
